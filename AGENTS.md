# AGENTS.md

## 项目协作规范

### 代码清理规范

1. 功能下线后，代码中不能保留废弃实现。
2. 不保留废弃组件、废弃页面、废弃接口调用、废弃状态、废弃样式和未使用依赖。
3. 不保留仅用于兼容历史方案的过渡代码；完成迁移后必须移除。
4. 每次重构或功能替换完成后，应同步清理无用代码，保证仓库可维护性。

### 默认执行约定

1. 后续开发默认遵循“代码不留废弃实现，文档保留版本轨迹”的规则。
2. 若用户明确要求保留某段历史代码或实验性实现，需单独说明原因和范围。

### 规范沉淀约定

1. 后续在项目协作过程中新增的规范、约束、默认流程和长期有效约定，均需同步写入 `AGENTS.md`。
2. 处理新规范时，除在当前对话中执行外，还应更新 `AGENTS.md` 作为项目级长期记忆。
3. 若新规范与已有规范冲突，应以最新经用户确认的规则为准，并在 `AGENTS.md` 中明确覆盖关系或版本说明。

### 线程生效范围约定

1. `AGENTS.md` 中的项目规则默认作用于该项目的所有后续线程、子任务和协作轮次，而不只限于当前对话。
2. 当在本项目中开启新线程时，仍应继续遵循 `AGENTS.md` 中最近一次确认后的规则，尤其是交互规范。
3. 若用户在当前线程中明确补充或修改长期规则，该规则自确认时起立即覆盖当前线程后续内容，并作为项目级规则同步写入 `AGENTS.md`。
4. 已经发送出的历史消息不做回写修改，但从规则确认后的下一条响应开始，必须按最新规则执行。

### 交互规范

1. 语言要求：
   - 全部使用中文与我交流（包括解释、注释、生成内容）
   - 不要切换到英文，除非我明确要求
2. 行为方式：
   - 不要凭空假设不存在的代码
   - 优先从已有代码推导设计
   - 输出要结构化、清晰、工程化
3. 优先级要求：
   - 交互规范属于项目级高优先级规则
   - 后续该项目所有线程默认严格遵循交互规范
   - 若出现与历史习惯或默认行为不一致的情况，以 `AGENTS.md` 中当前交互规范为准
